¿Cómo funciona el Polimorfismo aquí?

En este simulador, cada clase de personaje se modela usando Programación Orientada a Objetos en JavaScript.

1
Herencia Común: Todos heredan de la clase base Character. Esto significa que todos tienen la propiedad stats y los métodos executeAction() y takeDamage().
2
Sobrescritura de Métodos: Cada personaje redefine (sobrescribe) estos métodos.
  • El Guerrero sobrescribe takeDamage para aplicar su 40% de bloqueo con escudo.
  • El Mago sobrescribe executeAction para validar y consumir maná al lanzar hechizos mágicos.
  • El Clérigo y la Curandera sobrescriben executeAction para curar aliados en lugar de dañar enemigos.
  • El Berserker sobrescribe takeDamage para aumentar su ataque al perder salud.
3
Llamada Polimórfica: Durante el combate, el motor de juego ejecuta la misma línea de código: 
personajeActivo.executeAction(habilidadIndex, objetivo)
El motor no sabe ni le importa el tipo exacto del personaje. JavaScript busca dinámicamente y ejecuta el comportamiento correspondiente a la subclase específica del personaje. ¡Eso es polimorfismo!

Esta es la clase base Character de la que heredan todos los personajes de la arena.


class Character {
    constructor(data) {
        this.id = data.id;
        this.nombre = data.nombre;
        this.name = data.name;
        this.stats = { ...data.stats };
        this.maxHp = data.stats.hp;
        this.maxMana = data.stats.mana;
        this.hp = data.stats.hp;
        this.mana = data.stats.mana;
        this.habilidades = data.habilidades;
        this.imagenes = data.imagenes;
    }

    // Método Base de Daño
    takeDamage(amount, type = "physical") {
        let finalDamage = Math.max(1, amount - this.stats.defense);
        this.hp = Math.max(0, this.hp - finalDamage);
        return { damage: finalDamage, absorbed: 0, effects: [] };
    }

    // Método Base de Acción
    executeAction(abilityIndex, target) {
        const ability = this.habilidades[abilityIndex];
        const damage = ability.base_damage || ability.danio_base || 0;
        
        // Llamada polimórfica a takeDamage en el objetivo
        const result = target.takeDamage(damage, "physical");
        return {
            abilityName: ability.nombre,
            text: `${this.nombre} usa ${ability.nombre} sobre ${target.nombre} causando ${result.damage} de daño físico.`
        };
    }
}

Entendiendo el Polimorfismo

El Polimorfismo (del griego poly = muchas, morph = formas) es uno de los pilares de la programación orientada a objetos.

En este Proyecto:

  • Una sola interfaz: Las acciones del juego interactúan con los personajes mediante un método común: personaje.executeAction().
  • Múltiples comportamientos: Cuando se presiona una habilidad, no hacemos condicionales complejos como: 
    if (esMago) { lanzarHechizo() } else if (esGuerrero) { usarEscudo() }
    En su lugar, simplemente llamamos al método común y dejamos que cada objeto responda según su propia clase heredada.

Beneficios:

  1. Extensibilidad: Si quisiéramos añadir una nueva clase, por ejemplo un Arquero o un Nigromante en el futuro, solo necesitaríamos crear una nueva subclase sin modificar el código de la arena o del simulador.
  2. Limpieza de Código: Evita largas y confusas cadenas de switch o if/else.
  3. Encapsulación: Cada personaje gestiona sus propias reglas de combate, daños y recursos.

      Conexión con la API DevCop95

      ¿Cómo se conecta este simulador con la API?

      Esta aplicación interactiva consume datos dinámicos en formato JSON directamente del repositorio público de DevCop95/rpgdevAPI. A continuación se explica el proceso técnico paso a paso:

      1
      Endpoints consumidos:

      Primero, se lee el índice general index.json para verificar el estado de los personajes. Luego, se realiza un fetch() a cada archivo JSON de personaje:

      https://raw.githubusercontent.com/DevCop95/rpgdevAPI/main/data/{personaje}.json
      2
      Estructura del JSON:

      El servidor devuelve un objeto con propiedades del personaje (stats, habilidades, imágenes, historia, fortalezas, debilidades):

      {
  "id": "guerrero",
  "nombre": "Guerrero",
  "titulo": "Defensor Inquebrantable",
  "stats": { "hp": 1100, "mana": 80, "defense": 35 },
  "fortalezas": [ "Alta Supervivencia", "Mitigación con Escudo" ],
  "habilidades": [ { "nombre": "Muro de Escudos", "danio_base": 0 } ]
}
      3
      Procesamiento en JavaScript (Polimorfismo):

      Una vez descargados los datos con fetch(), pasamos el JSON a nuestro CharacterFactory:

      const personaje = CharacterFactory.create(data);

      El Factory analiza el data.id e instanciará la subclase correspondiente (Warrior, Mage, etc.). Esto permite que cada personaje sobrescriba sus propios métodos de combate, aplicando Polimorfismo puro en tiempo de ejecución.

      RPG OOP Arena

      ¡Bienvenido al Simulador de Polimorfismo!

      Esta aplicación interactiva consume datos en tiempo real de la API de personajes RPG de DevCop95/rpgdevAPI para explorar de forma didáctica los principios de Programación Orientada a Objetos (OOP) y Polimorfismo en JavaScript.

      Las Clases de la Arena

      Guerrero El defensor principal. Sobrescribe la defensa para bloquear daño físico con su escudo.
      Berserker Furia desatada. Su daño escala de forma polimórfica a medida que pierde salud.
      Mago Hechicero. Consume maná para lanzar potentes conjuros mágicos que ignoran defensa física.
      Clérigo Soporte híbrido. Cura a sus aliados y conjura escudos de luz de absorción temporal.
      Curandera Sanadora pura. Su curación tiene un bono de salud crítica si el objetivo tiene menos de 35% HP.
      Ninja Ágil y mortal. Tiene probabilidad pasiva de esquivar ataques y ejecuta dobles golpes críticos.
      Templario Paladín sagrado. Absorbe daño con barreras sagradas y refleja el 20% del daño de vuelta.
